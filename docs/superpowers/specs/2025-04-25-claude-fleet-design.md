# Claude Fleet - Session 管理工具设计文档

## 项目概述

Claude Fleet 是一个管理多个 Claude Code session 的桌面应用工具。提供统一界面查看、管理和快速切换多个 Claude Code 实例，并在需要时恢复已关闭的 session。

### 目标用户

同时运行 3-10 个 Claude Code session 的开发者，session 分布在不同项目目录，需要监控状态、快速切换、查找历史记录。

### 核心价值

- **状态监控**：实时显示运行中 session 状态，等待输入时主动提示
- **快速切换**：一键跳转到对应的 Windows Terminal 窗口
- **历史管理**：收藏、搜索、恢复历史 session

---

## 页面结构

应用包含两个独立 Tab：

### Tab 1: "运行中"

监控当前运行的所有 Claude Code session。

**功能要点：**
- 卡片列表展示运行中的 session
- 显示：名称、路径、状态、上次活动时间
- 状态标签：运行中（绿色）、等待输入（黄色高亮）、已完成（灰色）
- 等待输入的 session 用醒目边框和背景高亮
- "跳转到终端"按钮：自动定位并激活对应的 Windows Terminal 窗口
- 顶部搜索栏：搜索名称、路径

### Tab 2: "Session 管理"

收藏/历史 session 的查看、恢复、管理。

**左右分栏布局：**

#### 左侧（列表区，280px 固定宽度）

- Session 列表，选中项高亮显示
- 搜索栏：搜索名称、路径、对话内容（模糊匹配）
- "仅显示收藏"开关（默认开启）
- 时间筛选下拉框（显示全部时出现，默认近30天）
- 星号图标：实心★已收藏 / 空心☆未收藏，点击切换状态
- 列表/目录视图切换按钮
- 新建 Session 按钮（+）
- 独立滚动

#### 右侧（详情区）

- 名称编辑：输入框 + 保存按钮
- 元数据：路径、Session ID、状态、创建时间、上次活动、对话轮数
- 恢复命令显示（可一键复制）
- 完整对话历史（滚动查看）
- 操作按钮：恢复、收藏切换、删除（仅未收藏时显示）
- 独立滚动

#### 目录视图（切换选项）

- 按工作目录路径树状展开
- 每个目录节点显示 session 数量
- 状态颜色指示：绿色运行中、黄色等待输入、灰色已完成
- 等待输入的 session 黄色背景高亮
- 紧凑的操作按钮

## 功能设计

### 1. 新建 Session

**交互方式：** 弹窗（居中显示）

**弹窗内容：**
- 工作目录选择
  - 输入框：手动输入路径
  - "浏览..."按钮：打开系统文件夹选择对话框
  - 常用路径快捷按钮（可配置）
- Session 名称（可选，默认使用目录名称）
- "启动 Claude Code"按钮

**行为：**
- 点击启动 → 打开新的 Windows Terminal 窗口
- 在指定目录启动 Claude Code
- 新建 session 自动加入收藏

### 2. 恢复 Session

**行为：**
- 点击"恢复"按钮 → 启动新的 Windows Terminal 窗口
- 自动执行 `claude --resume <session-id>` 命令
- 在对应工作目录启动

### 3. 跳转到终端

**行为：**
- 点击"跳转到终端"按钮
- 自动定位对应的 Windows Terminal 窗口
- 激活并置顶该窗口

**前提：** 用户使用 Windows Terminal，每个 session 一个独立窗口（非多标签页模式）

### 4. 状态提示

**触发时机：** session 从其他状态变为"等待输入"时

**提示方式：**
- 声音提示：播放提示音
- 桌面通知：弹出 Windows 系统通知，点击通知可跳转到终端

**生效范围：** 仅应用运行时生效（关闭应用后通知停止）

### 5. 收藏管理

- 点击星号图标切换收藏状态
- 实心★表示已收藏，空心☆表示未收藏
- 默认显示仅收藏的 session
- 关闭开关可显示全部历史 session

### 6. 删除功能

- 删除按钮仅在未收藏的 session 上显示
- 需先取消收藏才能删除
- 删除时将 session 记录从 Claude 原生存储文件夹中移除

### 7. 搜索功能

**搜索范围：** 名称、工作目录路径、对话内容（模糊匹配）

**适用场景：**
- "运行中" Tab：搜索名称、路径
- "Session 管理" Tab：搜索名称、路径、对话内容

### 8. 时间筛选

**位置：** Session 管理 Tab，左侧列表区

**显示条件：** 关闭"仅显示收藏"开关后出现

**默认值：** 近30天

**可选范围：** 近3天、近7天、近30天、全部时间

## 技术选型

### 前端框架
- React + TypeScript
- Tailwind CSS + shadcn/ui

### 桌面应用框架
- Tauri（推荐）或 Electron
- 需要能力：窗口管理、系统通知、进程控制、文件系统访问

### 状态监控（实现方案）
- 通过 Claude Code 全局钩子通知应用
- 钩子触发时机：启动 session、进入等待用户输入状态
- 应用接收钩子通知后更新 UI 并触发提示

### 数据来源
- Claude session 数据存储位置：需调研 Claude Code 原生 session 存储路径
- 应用自有数据：收藏列表、常用路径配置、自定义名称映射

## 用户交互流程

### 典型使用场景

**场景 1：监控运行中的 session**
1. 打开应用，查看"运行中" Tab
2. 看到所有运行中 session 的状态
3. 某 session 进入"等待输入" → 收到声音 + 桌面通知
4. 点击通知或"跳转到终端"按钮 → Windows Terminal 窗口置顶
5. 在终端中输入回复，继续工作

**场景 2：恢复历史 session**
1. 切换到"Session 管理" Tab
2. 搜索或浏览找到目标 session
3. 点击"恢复"按钮
4. 新终端窗口打开，自动执行恢复命令
5. 继续之前的对话

**场景 3：新建 session**
1. 点击左上角"+"按钮
2. 弹窗中选择工作目录（输入或浏览）
3. 点击"启动 Claude Code"
4. 新终端窗口打开，在指定目录启动 Claude
5. session 自动加入收藏

**场景 4：查找历史对话**
1. 切换到"Session 管理" Tab
2. 关闭"仅显示收藏"开关
3. 在搜索框输入关键词（对话内容模糊搜索）
4. 点击匹配的 session，右侧显示完整对话历史
5. 滚动查看或复制相关信息

## 非功能需求

### 性能
- session 列表加载：支持大量历史记录（数百条）
- 状态更新：实时响应钩子通知
- 搜索响应：毫秒级返回结果

### 可靠性
- 钩子通知丢失时提供降级方案（如手动刷新）
- 删除操作需确认提示
- 异常情况友好提示

### 可扩展性
- 常用路径可配置
- 未来可扩展为常驻后台（托盘应用）
- 支持其他终端应用（如 VS Code 终端）
